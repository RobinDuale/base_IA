# Vellum

> Vellum est une plateforme de développement (LLMOps) conçue pour aider les équipes techniques à construire, tester, évaluer et déployer des applications basées sur des modèles de langage (LLM). Elle centralise la gestion des prompts, le versioning et le monitoring pour optimiser la performance et la fiabilité des applications IA.

## Résumé

- Type : Outil IA et No-Code
- Catégorie : Productivité
- Niveau : Avancé
- Gratuité : Version gratuite limitée disponible pour les développeurs individuels et les petits projets (Hobby tier).

## Quand utiliser Vellum

Dès que vous passez de l'expérimentation locale d'un LLM à une application de production nécessitant une maintenance, une évaluation rigoureuse et une traçabilité des coûts.


## Pourquoi utiliser Vellum

Gestion centralisée des prompts et versioning - Outils de test et d'évaluation intégrés (Evals) - Monitoring et analyse des performances en temps réel - Support multi-modèles (OpenAI, Anthropic, Cohere, modèles open source) - Réduction du temps de mise sur le marché des applications IA


## Limites de Vellum

Courbe d'apprentissage orientée développeur - Nécessite des compétences techniques pour une intégration optimale - Coût peut devenir élevé à grande échelle



## Exemples et workflows

Créer un prompt -> Tester avec une suite de données de test -> Comparer les résultats -> Déployer via API - Capturer les feedbacks utilisateurs -> Analyser les erreurs dans Vellum -> Ajuster le prompt -> Redéployer


## Modèle économique

SaaS (Freemium pour les petites équipes / Payant avec niveaux d'entreprise)


## Quand payer

Dès que les besoins dépassent les quotas gratuits, pour accéder aux fonctionnalités de collaboration en équipe, à l'authentification SSO, à un support dédié ou à des limites de volume API plus élevées.


## Alternatives

LangSmith - Weights & Biases - PromptLayer - Helicone


## Complémentaire avec

LangChain - OpenAI API - Anthropic API - Pinecone - Supabase


## Scénario débutant

Utiliser Vellum pour gérer et itérer sur un prompt système simple afin d'améliorer la consistance des réponses d'un chatbot client.


## Scénario intermédiaire

Mettre en place une batterie de tests (Evals) sur Vellum pour vérifier que les changements de prompt n'altèrent pas les performances sur un jeu de données spécifique avant la mise en prod.


## Scénario avancé

Utiliser Vellum pour monitorer en temps réel les appels API d'une application multi-agents, détecter les dérives (drift) et ajuster dynamiquement les configurations des modèles utilisés en production via le SDK.
